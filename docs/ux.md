# UX-концепция

## Целевая аудитория

Активные интернет-пользователи 20–40 лет, у которых от 5 до 20 регулярных подписок (стриминги, музыка, облака, рабочие тулы, образовательные платформы), и которые хотят навести порядок в автосписаниях

## Персоны

### Персона 1: Аня, 26, дизайнер-фрилансер

- **Контекст:** ведёт несколько проектов параллельно, у неё накопилось ~12 платных подписок (Figma, Adobe CC, Notion, Spotify, Netflix, домен, хостинг портфолио, AI-тулы, …). Не помнит точный итог по месяцу — догадывается «где-то 8000 ₽».
- **Цели:**
  1. Увидеть итоговую сумму в месяц по каждой категории и в целом.
  2. Понять, где можно срезать — какие сервисы дублируют функции.
- **Важно в первые 30 секунд:** на главной — конкретная цифра «расход в месяц», список ближайших списаний, чтобы быстро решить «отписываться или оставить».
- **Стоп-факторы:** требование зарегистрироваться, прежде чем посмотреть «как выглядит интерфейс»; необходимость вручную вводить 12 подписок без шаблонов.

### Персона 2: Игорь, 32, тимлид в IT

- **Контекст:** оплачивает рабочие подписки на команду (GitHub Copilot, Linear, Notion, JetBrains All Products Pack, серверы), плюс ~5 личных. Часть подписок годовые, часть месячные — забывает, когда что списывается.
- **Цели:**
  1. Установить лимиты на категории «Работа» и «Утилиты», чтобы не выйти за бюджет команды.
  2. Получать уведомления заранее — за 3–7 дней до крупного годового списания (Copilot, JetBrains).
- **Важно в первые 30 секунд:** дашборд с предупреждением «превышение лимита» если оно есть; календарный обзор ближайших списаний по сумме.
- **Стоп-факторы:** интерфейс, где CRUD требует много кликов; невозможность отделить рабочие и личные подписки (решается категориями).

## User Stories

1. **Как Аня**, хочу видеть итоговый расход в месяц одним числом, чтобы за 5 секунд понять масштаб трат.
   - Шаги: открыть приложение → залогиниться (или войти по демо) → главный экран показывает stat-card «Расход в месяц».
   - Done-критерий: на дашборде в первой карточке отображается сумма всех активных подписок, нормализованная к месячной (yearly ÷ 12, quarterly ÷ 3, weekly × 4.345), форматированная с символом валюты.

2. **Как Аня**, хочу удалять подписку в один-два клика, чтобы не разбираться с многошаговыми диалогами.
   - Шаги: список подписок → корзина у конкретной карточки → подтверждение в браузерном confirm() → карточка исчезает.
   - Done-критерий: после удаления список локально пересчитан, общий расход на дашборде уменьшен на её эквивалентный месячный, mock-API получил DELETE-запрос.

3. **Как Игорь**, хочу установить лимит на категорию «Стриминг», чтобы система предупреждала когда я его превышаю.
   - Шаги: «Категории» → найти карточку «Стриминг» → ввести 1000 в поле «Лимит в месяц» → зафокусить вне поля.
   - Done-критерий: при сумме активных стриминг-подписок > 1000 ₽/мес карточка категории получает красную рамку, прогресс-бар целиком красный, появляется строка «Превышение лимита: NN ₽», на дашборде stat-card «Превышение лимита» инкрементируется.

4. **Как Игорь**, хочу настроить уведомление за N дней до списания каждой подписки персонально.
   - Шаги: создать/редактировать подписку → поле «Уведомить за N дней» → сохранить.
   - Done-критерий: на дашборде в блоке «Ближайшие списания» появляются только подписки, у которых до даты ≤ их `notifyDaysBefore`.

5. **Как преподаватель/проверяющий**, хочу зайти на публичную ссылку и за 30 секунд увидеть все ключевые сценарии без регистрации.
   - Шаги: открыть `https://neckser.github.io/Risu/` → нажать «Заполнить демо-данными» → войти.
   - Done-критерий: загружается дашборд с 5 предзагруженными подписками (Netflix, Spotify, PlayStation Plus, iCloud, GitHub Copilot), 4 stat-card, ближайшие списания (iCloud завтра, Netflix через 5 дней).

## Дизайн-система (Taiga UI 5)

### Используемые компоненты Taiga UI 5

1. **`TuiButton`** — все CTA: «Войти», «Создать», «Сбросить», «Удалить подписку», ссылки-кнопки в шапке «Добавить».
2. **`TuiIcon`** — декоративные иконки в sidebar-навигации, в stat-картах, в карточках категорий, в кнопках действий.
3. **`TuiRoot`** — корневой контейнер приложения, обеспечивает портал-рендер всплывающих элементов (тосты).
4. **`TuiAlertService` + `TuiToastService`** — toast-уведомления об ошибках API в правом верхнем углу.
5. **`TuiBadgeNotification`** — счётчик ближайших списаний на пункте меню «Дашборд» в sidebar.
6. **`provideTaiga` + `provideEventPlugins`** — глобальная конфигурация (кастомные скроллбары, event-зоны вне Angular для производительности).

### Кастомные компоненты

- **`AppShellComponent`** (`core/layout/`) — sidebar-навигация с активной подсветкой пункта, бейджем уведомлений, инфой о пользователе и кнопкой выхода. Адаптивен: на ≤720px sidebar превращается в горизонтальное меню сверху.
- **`AppAuthShellComponent`** (`features/auth/`) — центрированный layout 380px для login/register с фирменным брендингом и градиентным фоном.
- **`StatCardComponent`** (`shared/components/stat-card/`) — переиспользуемая карточка статистики с label, иконкой, hint и контентом через `<ng-content>`. Использует сигнальные `input.required<>()` / `input<>()`.
- **`PageHeaderComponent`** (`shared/components/page-header/`) — единый заголовок страницы с title, subtitle и слотом для actions.
- **`SubscriptionCardComponent`** (`features/subscriptions/components/`) — карточка подписки в списке: аватар по первой букве, название, категория, дата следующего списания, цена, кнопки «Пауза»/«Удалить». Сигнальные `input` и `output`.

### CSS-токены и form-примитивы

- Все цвета берутся из CSS-переменных Taiga design-tokens (`--tui-text-action`, `--tui-background-base`, `--tui-status-negative`, `--tui-border-normal`).
- Глобальные form-классы в `src/styles.scss`: `.field`, `.field__label`, `.field__input`, `.field__select`, `.field__textarea`, `.field__check`, `.field__row`, `.field__error` — нативные HTML-элементы, стилизованные под общую визуальную систему. Состояния `:focus`, `:disabled`, `.ng-invalid.ng-touched` обработаны.

## Приоритеты экранов

### Первый экран (must-see)
- **Login** — single-button shortcut «Заполнить демо-данными» для проверяющих.
- **Dashboard** — основной экран после входа: 4 stat-card (расход в месяц, расход в год, скоро списания, превышение лимита) + два панели (ближайшие списания и топ категорий) выше fold'а на десктопе.

### Вторично
- **Subscriptions list** — глубокий разбор каталога с поиском/фильтром/сортировкой.
- **Subscription edit** — форма создания и редактирования (один компонент для обоих режимов через `:id` параметр).
- **Categories** — управление лимитами с прогресс-барами.

### Сознательно убрано
- **Push-уведомления через Web Push API** — требует service worker и PushSubscription, что значительно усложняет архитектуру. Reminder реализован как in-app бейдж на дашборде.
