# План разработки

## Цель проекта

**Risu** — это веб-приложение для отслеживания подписок и регулярных платежей. Продукт решает проблему забытых автосписаний: пользователь часто не помнит, сколько именно он платит за стриминги, облачные хранилища, рабочие тулы и т.п., и обнаруживает истинную сумму расходов, только когда списание уже произошло.

Приложение позволяет завести каталог подписок с указанием стоимости, периодичности и даты следующего платежа, категоризировать их (стриминг / игры / утилиты / работа / образование / прочее), устанавливать лимиты на категории и видеть напоминания о ближайших списаниях. Все данные пользователя изолированы по аккаунту через токен-авторизацию; mock-сервер на стороне клиента симулирует backend, что позволяет деплоить приложение как полностью статический SPA на GitHub Pages без отдельного хостинга API.

## Пользовательские сценарии

1. **Регистрация и заведение каталога**
   Новый пользователь создаёт аккаунт через `/auth/register`, попадает на пустой `/dashboard`, переходит в раздел Подписки, добавляет 3–5 своих регулярных платежей через форму с валидацией. После сохранения видит на дашборде агрегированный месячный расход, годовую проекцию и ближайшее списание.

2. **Контроль расходов через лимиты**
   Существующий пользователь логинится, идёт в раздел «Категории», ставит лимит 1000 ₽/мес на «Стриминг». При превышении (Netflix + Spotify + YouTube > 1000) карточка категории подсвечивается красным с конкретной суммой превышения, а на дашборде в карточке «Превышение лимита» появляется счётчик.

3. **Поиск и пауза подписки**
   Пользователь забыл, как назывался его малоизвестный сервис, - находит его через поиск по названию на `/subscriptions`, фильтрует по категории Утилиты, ставит сортировку По цене (дороже) чтобы переоценить топовые позиции, ставит ненужный сервис на паузу через кнопку Пауза (без удаления, чтобы вернуться позже). Метрика общего расхода тут же пересчитывается без перезагрузки.

4. **Многоустройственный демо-доступ**
   Преподаватель открывает публичную ссылку, нажимает «Заполнить демо-данными» на форме логина, попадает в готовый кабинет с 5 предзагруженными подписками и сразу проверяет все сценарии без регистрации.

## Этапы разработки

| Срок | Задача |
|------|--------|
| До конца апреля | Базовая структура проекта (Angular 21 + Taiga UI 5 + NgRx Signal Store), компоненты `core/shared/features`, формы логина/регистрации с валидацией, модель данных (`Subscription`, `Category`, `User`), routing с lazy loading, базовые гарды, mock API на HttpInterceptor с seed-данными. |
| До конца мая | Полный CRUD подписок, дашборд со статистикой, категории с лимитами, поиск/фильтр/сортировка, авторизация с токенами в localStorage, error-interceptor с тостами Taiga, юнит-тесты на Vitest, E2E на Playwright, CI/CD-pipeline на GitHub Actions, деплой на GitHub Pages, ESLint + Stylelint + Prettier. Финальная полировка UX, прогон Lighthouse, финальное README с публичным URL, проверка чек-листа сдачи|


## Риски

**Локальное хранение данных и иллюзия многоустройственности** — пользователь может ожидать, что аккаунт переедет на телефон. \
В README и на странице настроек явно пояснено, что mock-сервер живёт в `localStorage` браузера; для проверки на нескольких устройствах на каждом доступен общий демо-аккаунт `demo@risu.app/demo1234`, который сидится при первом заходе.
